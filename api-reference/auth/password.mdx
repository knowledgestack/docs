---
title: "Password Auth"
description: "Create password-based accounts, sign in, manage sessions, and reset passwords."
---

Password-based authentication uses email and password credentials. The typical new-user flow is: send an email verification → create the user account → sign in.

---

## Send email verification

<br />

`POST https://api-staging.knowledgestack.ai/v1/auth/pw/email_verification`

Send a verification email to the provided address. The email contains a token that you pass to the [create user](#create-user) endpoint. This is the first step when registering a new password-based user.

### Request body

<ParamField body="email" type="string" required>
  The email address to verify. Must be a valid email format.
</ParamField>

### Response

Returns an `EmailSentResponse` on success.

<ResponseField name="email_id" type="string (uuid)">
  ID of the sent email message.
</ResponseField>

### Example

```bash
curl -X POST https://api-staging.knowledgestack.ai/v1/auth/pw/email_verification \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com"}'
```

```json
{
  "email_id": "3f2a1b4c-0000-0000-0000-000000000000"
}
```

## Create user

<br />

`POST https://api-staging.knowledgestack.ai/v1/auth/pw/user`

Create a new password-based user account. You must complete [email verification](#send-email-verification) first and supply the token from that email.

### Request body

<ParamField body="password" type="string" required>
  The user's password. Minimum 8 characters.
</ParamField>

<ParamField body="email_token" type="string" required>
  The email verification token received from the verification email.
</ParamField>

<ParamField body="first_name" type="string">
  User's first name (optional).
</ParamField>

<ParamField body="last_name" type="string">
  User's last name (optional).
</ParamField>

### Response `201`

Returns a `UserResponse` on success.

<ResponseField name="id" type="string (uuid)">
  User ID.
</ResponseField>
<ResponseField name="email" type="string">
  User's email address.
</ResponseField>
<ResponseField name="first_name" type="string">
  First name.
</ResponseField>
<ResponseField name="last_name" type="string">
  Last name.
</ResponseField>
<ResponseField name="idp_type" type="string">
  Identity provider type. One of `PASSWORD`, `GOOGLE`, `TENANT`.
</ResponseField>
<ResponseField name="current_tenant_id" type="string (uuid)">
  The tenant the user is currently scoped to.
</ResponseField>
<ResponseField name="current_tenant_role" type="string">
  The user's role in the current tenant.
</ResponseField>
<ResponseField name="default_tenant_id" type="string (uuid)">
  The tenant the user is signed into by default.
</ResponseField>

### Example

```bash
curl -X POST https://api-staging.knowledgestack.ai/v1/auth/pw/user \
  -H "Content-Type: application/json" \
  -d '{
    "password": "s3cur3p@ss",
    "email_token": "<token-from-email>",
    "first_name": "Ada",
    "last_name": "Lovelace"
  }'
```

## Sign in

<br />

`POST https://api-staging.knowledgestack.ai/v1/auth/pw/signin`

Validate email and password credentials. On success, the server sets a `ks_uat` session cookie containing the user access token and returns the user record.

### Request body

<ParamField body="email" type="string" required>
  The user's email address.
</ParamField>

<ParamField body="password" type="string" required>
  The user's password.
</ParamField>

<ParamField body="tenant_id" type="string (uuid)">
  Target tenant to sign into. If omitted, the user's default tenant is used.
</ParamField>

### Response `200`

Returns a `UserResponse`. See [Create user](#create-user) for field descriptions.

<Note>
  The `ks_uat` cookie returned by this endpoint is your user access token (UAT). Pass it as `Authorization: Bearer <uat>` on subsequent requests, or let the browser send it automatically via the cookie.
</Note>

### Example

```bash
curl -X POST https://api-staging.knowledgestack.ai/v1/auth/pw/signin \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "s3cur3p@ss"
  }'
```

## Send password reset email

<br />

`POST https://api-staging.knowledgestack.ai/v1/auth/pw/send_reset_email`

Send a password reset email to the given address. The email contains a token that you pass to [reset with token](#reset-password-with-token).

### Request body

<ParamField body="email" type="string" required>
  The email address associated with the account.
</ParamField>

### Response `200`

Returns an `EmailSentResponse`.

<ResponseField name="email_id" type="string (uuid)">
  ID of the sent email message.
</ResponseField>

### Example

```bash
curl -X POST https://api-staging.knowledgestack.ai/v1/auth/pw/send_reset_email \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com"}'
```

## Reset password

<br />

`POST https://api-staging.knowledgestack.ai/v1/auth/pw/reset`

Reset the password for the currently authenticated user using their existing password. Requires a valid UAT.

### Request body

<ParamField body="old_password" type="string" required>
  The user's current password.
</ParamField>

<ParamField body="new_password" type="string" required>
  The new password. Minimum 8 characters.
</ParamField>

### Response `200`

Returns a `UserResponse` reflecting the updated user.

### Example

```bash
curl -X POST https://api-staging.knowledgestack.ai/v1/auth/pw/reset \
  -H "Authorization: Bearer <your-api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "old_password": "s3cur3p@ss",
    "new_password": "n3wp@ssw0rd"
  }'
```

## Reset password with token

<br />

`POST https://api-staging.knowledgestack.ai/v1/auth/pw/reset_with_token`

Reset the password using an email verification token from the [send reset email](#send-password-reset-email) flow. This does not require an existing session.

### Request body

<ParamField body="email_token" type="string" required>
  The password reset token from the reset email.
</ParamField>

<ParamField body="new_password" type="string" required>
  The new password. Minimum 8 characters.
</ParamField>

### Response `200`

Returns a `UserResponse`.

### Example

```bash
curl -X POST https://api-staging.knowledgestack.ai/v1/auth/pw/reset_with_token \
  -H "Content-Type: application/json" \
  -d '{
    "email_token": "<token-from-email>",
    "new_password": "n3wp@ssw0rd"
  }'
```

## Refresh UAT

<br />

`POST https://api-staging.knowledgestack.ai/v1/auth/uat`

Refresh the user access token or switch the active tenant. Call this when the current UAT has expired or when you want to operate under a different tenant.

### Query parameters

<ParamField query="tenant_id" type="string (uuid)">
  Target tenant ID to switch to. Omit to refresh the current tenant.
</ParamField>

### Response `200`

Returns a `UserResponse` with the updated `current_tenant_id`.

### Example

```bash
# Refresh current tenant
curl -X POST https://api-staging.knowledgestack.ai/v1/auth/uat \
  -H "Authorization: Bearer <your-api-key>"

# Switch tenant
curl -X POST "https://api-staging.knowledgestack.ai/v1/auth/uat?tenant_id=<tenant-id>" \
  -H "Authorization: Bearer <your-api-key>"
```

## Sign out

<br />

`POST https://api-staging.knowledgestack.ai/v1/auth/signout`

Invalidate the current session and clear the `ks_uat` cookie.

### Response `200`

Returns an empty JSON object `{}`.

### Example

```bash
curl -X POST https://api-staging.knowledgestack.ai/v1/auth/signout \
  -H "Authorization: Bearer <your-api-key>"
```
